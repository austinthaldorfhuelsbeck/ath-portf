Restaurant Reservation Manager

A lightweight and responsive full-stack application to manage your restaurant’s reservations and seating.

Demo Site

Repo

About the project

The user is presumed to be a host/hostess at a restaurant. The user can accept new reservations as they’re called in, search existing reservations, and manage where and when reservations are seated within the restaurant.

This is my submission for Chegg Learning’s Thinkful Software Engineering full-stack capstone project. I was provided with a set of user stories and a suite of end-to-end tests, and evaluated on my submission.

Project Execution

The frontend of this application is designed to be lightweight and user-friendly, therefore the development process began with the backend. Following a schedule to meet the deadline, I was able to develop and test the Express API fully before developing the frontend.

Utilizing Bootstrap allowed more time to focus on handling data and errors within React to ensure the application’s stability. React Router hooks are leveraged to create a dynamic and fluid presentation.

Project Results

Through following a set of user stories and unit tests, the development process was predictable, and it was rewarding to develop a project that not just works but meets a spec. By completing the project early, I was able to refactor and organize the code.

The assignment submission was accepted with a few notes about code quality and efficiency, which I was able to interpret and correct.
