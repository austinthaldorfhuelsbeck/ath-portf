About the project

When a colleague reached out about developing her personal website, she needed a finished product quickly in anticipation of her book release. By following self-made deadlines each step of the way, and through working closely with a graphic designer, I was able to deliver a stable deployment on-time.

After the client approved the design for the frontend, I got to work on developing a custom Node.js API. Working with a Postgres database, I was able to keep the data separated from the client, which allowed me to develop a second application to serve as an admin dashboard. From the admin dashboard, the user can manage the content on the site and publish new content.

Project Execution

Because this application is for a real-world client, the code requires maintenance and refactoring. To help refactor and debug quickly, as well as to help understand and document the code, this project utilizes TypeScript.

Because of the relational database structure, users are able to create and modify custom categories and tags for their content. The Express application handles validation and queries for a wide array of requests with Knex, allowing each frontend application to communicate with the server efficiently to locate the specific data required for each component of the application.

The dashboard is designed with reusable components to allow extension of functionality in the future. React Components are used to handle conditional styling with reusability.

In order to protect the data, access to both the Express application and the dashboard React application are protected with Auth0.

Project Results

The website cathyloerzel.com has been live for over two years, and has undergone several revisions to the design and layout. These revisions were accomplished utilizing software development lifecycle principles. Cathy is able to adjust the site’s content when needed.

The dashboard has multiple implementations running with different databases, to protect Cathy’s data while allowing a demo of the application.
